# MCP Benchmark EC2 Heavy

Tier-specific EC2 runtime for the heavy-tier MCP benchmark servers.
Regenerated by `mcp-benchmark/scripts/sync_ec2_repos.py` from the registry +
`sam/profiles/tiered.json` (local cold-spawn latency re-tiering, 2026-05-28).

## Heavy Tier Servers

1. `mcp-server-nationalparks`
2. `metmuseum-mcp`
3. `hugeicons-mcp-server`
4. `openfda-mcp`
5. `paper-search-mcp`
6. `openalex-mcp`
7. `pdf-reader-mcp`
8. `context7`
9. `json-query-mcp`
10. `fetch-readable-mcp`

## Key Files

- `ec2/heavy-route-config.json` - route config for `/heavy/mcp` and `/heavy/health`.
- `ec2/install-heavy.sh` - clones/builds the heavy-tier servers (GENERATED).
- `ec2/bootstrap-heavy-host.sh` - host deps + install + runtime build + PM2.
- `ec2/ecosystem.heavy.config.cjs` - PM2 process def using the route config.
- `ec2/runtime/` - Fastify streamable-HTTP to stdio MCP bridge.
- `ec2/adapters/` - vendored stdio adapters for this tier (GENERATED).

## EC2 Setup

```bash
bash ec2/bootstrap-heavy-host.sh
```

- MCP endpoint: `http://127.0.0.1:3000/heavy/mcp`
- Health endpoint: `http://127.0.0.1:3000/heavy/health`

`servers/` is empty in Git; the install flow populates it on the host at pinned commits.
